Інструкція користувача

Детальний посібник з використання системи "Менеджер задач".

1. Вхід у систему

Для входу виконайте наступні кроки:

  1. Перейдіть на сторінку входу: /login.
  2. Введіть ваші дані (ім'я користувача та пароль).
  3. Натисніть кнопку "Увійти".
  4. Після входу ви потрапите на сторінку зі списком задач.

Перейти до входу

2. Список задач

На цій сторінці відображаються задачі, в яких ви є виконавцем або автором.

  • Фільтри: Використовуйте бокову панель для фільтрації задач за статусом, автором або пріоритетом.
  • Пагінація: У таблиці відображаються по 7 задач на сторінці. Використовуйте кнопки "1", "2", "Далі" для переходу між сторінками.

Переглянути список задач

3. Перегляд задачі

Щоб переглянути деталі задачі, натисніть кнопку "Переглянути" у списку задач. Ви зможете:

  • Дізнатися основну інформацію про задачу: статус, дедлайн, відповідальні.
  • Переглянути та залишити коментарі в розділі "Листування".
  • Додати вкладені файли.

Приклад перегляду задачі

4. Створення задачі

Щоб створити нову задачу:

  1. Натисніть кнопку "Створити задачу" у верхньому меню.
  2. Заповніть всі необхідні поля: назву, опис, дедлайн тощо.
  3. Призначте виконавців задачі.
  4. Натисніть кнопку "Зберегти".

Створити задачу

5. Права доступу

Доступ до задач надається лише тим користувачам, які є виконавцями або авторами. Якщо у вас немає доступу, ви побачите повідомлення:

Увага: У вас немає доступу до цієї задачі.

Якщо ви вважаєте, що це помилка, зверніться до адміністратора.

6. Логіка статусів

У системі задач використовуються кілька основних статусів. Нижче наведено їх опис, логіку призначення та автоматичну поведінку системи:

  • Не розпочато:
    • Цей статус автоматично призначається при створенні задачі.
    • Означає, що виконавці ще не розпочали роботу над задачею.
    • Виконавець або автор може змінити статус на "Виконується".
  • Виконується:
    • Цей статус встановлюється вручну виконавцем, коли він/вона починає роботу над задачею.
    • Доступний для змін лише виконавцем або автором задачі.
    • Система автоматично оновлює статус, якщо завершення задачі перевищує дедлайн, змінюючи його на "Прострочено".
  • Завершено:
    • Цей статус встановлюється вручну виконавцем, коли задача виконана.
    • Після встановлення цього статусу задача недоступна для змін, але залишається доступною для перегляду.
    • Система автоматично записує час завершення задачі.
  • Прострочено (автоматичний):
    • Цей статус призначається автоматично системою, якщо задача не виконана до встановленого дедлайну.
    • Доступний лише для задач зі статусом "Виконується" або "Не розпочато".
    • Автор або виконавець повинен змінити статус вручну після завершення задачі.

Слідкуйте за статусами, щоб контролювати хід виконання задачі та вчасно отримувати результати. У випадках, коли статуси змінюються автоматично (наприклад, "Прострочено"), повідомлення з'являється у відповідному розділі задачі.
